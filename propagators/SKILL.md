# Propagators

## 目标

提供一套可复用流程：根据任务约束选择传播器、配置力模型、产出状态与误差结果，并给出可验证结论。

## 触发条件

当请求涉及以下内容时启用：
- propagator / propagation / 轨道传播
- 轨道预测、轨迹外推、状态外推
- TLE、SGP4、星历生成
- J2、气动阻力、太阳压、三体摄动
- Cowell 法、ODE 数值积分、长期仿真

## 默认流程

1. 明确场景约束  
   - 输入类型、中心天体、时长、精度、运行时约束
2. 选择传播器  
   - 两体 / J2 / SGP4 / 数值积分
3. 显式声明力模型  
   - 不允许“默认配置未说明”
4. 执行传播并产出结果  
   - 轨道状态、元素趋势、关键事件
5. 完成验证与标注  
   - 基线比较、误差统计、局限说明

## 传播器选择准则

### 两体（Kepler）
- 适用：短时快速估算、初步 sanity check
- 不适用：摄动敏感任务、长期趋势分析

### J2（或低阶摄动）
- 适用：需要 RAAN/近地点长期漂移趋势
- 典型：LEO/MEO 中期精度与效率平衡

### SGP4
- 适用：输入为 TLE，且目标是与编目体系保持一致
- 注意：TLE 本身存在时效与精度边界

### 数值积分（Cowell/ODE）
- 适用：高保真、定制力模型、灵敏度分析
- 代价：算力开销更高，参数配置更复杂

## 质量规则

- 单位必须明确（km/m、s/day）且全程一致
- 坐标系与历元必须显式给出
- 传播器选择必须有“约束 -> 结论”链路
- 力模型项必须完整列出，禁止隐含默认
- 结果必须至少有一个基线对比
- 结论必须区分“观测事实”与“推断判断”

## 快速检查清单

- 已识别输入与任务约束
- 已给出传播器选择与理由
- 已明确力模型与常数来源
- 已输出关键轨道结果与事件
- 已完成误差验证与局限标注
- 已提供可复现步骤
